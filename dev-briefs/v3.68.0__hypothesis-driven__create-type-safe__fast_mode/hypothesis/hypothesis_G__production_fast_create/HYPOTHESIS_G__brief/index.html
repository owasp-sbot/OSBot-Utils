Key Design Decisions¶

  1. Simplified Config - Only 2 flags:
  2. fast_create - Use schema-based creation

  3. skip_validation - Bypass __setattr__ validation

  4. Regular dict for cache - Classes never get GC'd at runtime, so WeakKeyDictionary is unnecessary overhead

  5. No __new__ override - Everything done in __init__, matching existing Type_Safe pattern

  6. Separate Schema Classes - Field__Schema and Class__Schema in dedicated files for clarity

  7. Module Singletons - type_safe_fast_create_cache and type_safe_step_fast_create follow existing patterns

Usage¶

from hypothesis_G__production_fast_create import Type_Safe__Config, Type_Safe__Hypothesis_G

class MyClass(Type_Safe__Hypothesis_G):
    name  : str = ''
    count : int = 0

# Normal creation (full validation)
obj = MyClass(name='test')

# Fast creation (~10-20x faster)
with Type_Safe__Config(fast_create=True):
    obj = MyClass(name='test')

# Fast everything (bulk loading)
with Type_Safe__Config(fast_create=True, skip_validation=True):
    for row in database_rows:
        obj = MyClass(**row)
