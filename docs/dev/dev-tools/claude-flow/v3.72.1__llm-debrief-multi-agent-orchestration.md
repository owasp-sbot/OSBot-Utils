# LLM Debrief: Multi-Agent Orchestration with Claude-Flow in Docker

> **A complete guide to implementing isolated, parallel AI agent workflows for content validation**

**version**: v3.72.1  

---

## Executive Summary

This document captures the design decisions, architecture, and implementation of a **multi-agent orchestration system** using Claude-Flow running in an isolated Docker environment. The solution addresses fundamental limitations in Claude's current consumer interfaces (Cowork/claude.ai) by enabling:

- **True parallel agent execution** with separate contexts
- **Executor/Reviewer two-pass workflows** for quality assurance
- **Complete isolation** from the host development environment
- **Persistent memory** across agent sessions
- **Web UI** for monitoring and interaction

## Usage Guide

### Initial Setup

```bash
# 1. Extract the package
unzip claude-flow-docker.zip
cd claude-flow-docker

# 2. Run setup
chmod +x setup.sh
./setup.sh

# 3. Configure API key
cp .env.example .env
nano .env  # Add your ANTHROPIC_API_KEY

# 4. Link your project (optional)
ln -sf /path/to/NotebookLM__Infographics-and-slides workspace

# 5. Build and start
docker-compose up -d --build
```

### Daily Usage

```bash
# Start environment
docker-compose up -d

# Enter container
docker-compose exec claude-flow bash

# Inside container:
claude-flow init                    # First time only
claude-flow start-ui                # Start Web UI (http://localhost:3000)

# Or run workflows directly:
./scripts/two-pass-workflow.sh "folder/path" "Create SEMANTIC-GRAPH.md"

# Exit container
exit

# Stop environment
docker-compose stop
```

### Running the Two-Pass Workflow

```bash
# Inside the container:

# Option 1: Using the helper script
./scripts/two-pass-workflow.sh \
    "_for_linkedin/to-publish/MyFolder" \
    "Create SEMANTIC-GRAPH.md"

# Option 2: Manual execution
claude-flow swarm init --topology hierarchical --max-agents 3

claude-flow agent spawn coder \
    --name "executor" \
    --task "Create SEMANTIC-GRAPH.md for MyFolder following .claude/create-semantic-graph.md"

claude-flow agent spawn reviewer \
    --name "reviewer" \
    --task "Validate MyFolder/SEMANTIC-GRAPH.md against .claude/review-checklists.md"

claude-flow swarm status
```

| Problem | Solution |
|---------|----------|

| Memory not persisting | Ensure `./data/` is mounted and writable |

## Conclusion

This implementation bridges the gap between Claude's consumer interfaces and enterprise-grade multi-agent orchestration. By running Claude-Flow in an isolated Docker environment, we achieve:

1. **True parallel agent execution** with separate contexts
2. **Genuine independent review** (Reviewer doesn't know Executor's reasoning)
3. **Complete isolation** from host system (security)
4. **Persistent memory** across sessions
5. **Scalable batch processing** for large content libraries

The two-pass Executor/Reviewer pattern, combined with Docker isolation, provides a production-ready workflow for quality-assured content generation at scale.
