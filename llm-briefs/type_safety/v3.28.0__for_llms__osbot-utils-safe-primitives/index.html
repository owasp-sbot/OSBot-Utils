OSBot-Utils Safe Primitives Complete Reference Guide¶

Version: v3.28.0
Created: 11/Nov/2025

Overview¶

OSBot-Utils provides a comprehensive collection of type-safe primitive classes that enforce validation rules at instantiation time. These primitives prevent invalid data from entering your system and provide clear error messages when validation fails.

Key Benefits: - Compile-time safety: Catch invalid data at creation time, not runtime - Self-documenting: Type annotations clearly communicate constraints - Zero performance overhead: Validation happens once at instantiation - Composable: Use in Type_Safe classes for complete type safety

Core Primitive Types¶

Safe_Int¶

Base class for type-safe integers with validation rules.

from osbot_utils.type_safe.primitives.core.Safe_Int import Safe_Int

class Age(Safe_Int):
    min_value = 0
    max_value = 150

age = Age(25)           # ✓ Valid
age = Age(-5)           # ✗ Raises ValueError
age = Age(200)          # ✗ Raises ValueError

Configuration Options: - min_value: Minimum allowed value (inclusive) - max_value: Maximum allowed value (inclusive) - allow_none: Whether None is allowed (default: True) - allow_bool: Whether bool values are allowed (default: False) - allow_str: Whether string conversion is allowed (default: True) - strict_type: If True, only accept int type (default: False) - clamp_to_range: Clamp to min/max instead of raising error (default: False)

Arithmetic Operations: All arithmetic operations maintain type safety and return instances of the same class:

result = Age(25) + 5    # Returns Age(30)
result = Age(100) * 2   # Returns Age(200) or raises if > max_value

Safe_UInt¶

Unsigned integer - only accepts non-negative values.

from osbot_utils.type_safe.primitives.core.Safe_UInt import Safe_UInt

class Count(Safe_UInt):
    max_value = 1000

count = Count(50)       # ✓ Valid
count = Count(-1)       # ✗ Raises ValueError (min_value is 0)

Configuration: - Inherits all Safe_Int options - min_value automatically set to 0

Safe_Float¶

Base class for type-safe floats with precise decimal handling.

from osbot_utils.type_safe.primitives.core.Safe_Float import Safe_Float

class Price(Safe_Float):
    min_value = 0.0
    decimal_places = 2
    use_decimal = True    # Use Decimal for exact arithmetic

price = Price(19.99)
price = Price("19.99")   # String conversion supported

Configuration Options: - min_value: Minimum allowed value - max_value: Maximum allowed value - allow_none: Whether None is allowed (default: True) - allow_bool: Whether bool values allowed (default: False) - allow_str: Whether string conversion allowed (default: True) - allow_int: Whether int conversion allowed (default: True) - allow_inf: Whether infinite values allowed (default: False) - strict_type: If True, only accept float type (default: False) - decimal_places: Number of decimal places (default: None) - use_decimal: Use Python Decimal for exact arithmetic (default: True) - epsilon: Tolerance for equality comparisons (default: 1e-9) - round_output: Round to decimal_places (default: True) - clamp_to_range: Clamp to min/max range (default: False)

Why use_decimal=True?

# Standard Python float issues:
0.1 + 0.2 == 0.3  # False! (floating point error)

# With Safe_Float and use_decimal=True:
class Money(Safe_Float):
    decimal_places = 2
    use_decimal = True

Money(0.1) + Money(0.2) == Money(0.3)  # True!

Arithmetic Operations: All operations preserve type safety:

price1 = Price(10.50)
price2 = Price(5.25)
total = price1 + price2    # Returns Price(15.75)

Safe_Str¶

Base class for type-safe strings with regex validation and sanitization.

from osbot_utils.type_safe.primitives.core.Safe_Str import Safe_Str
import re

class Username(Safe_Str):
    regex = re.compile(r'[^a-zA-Z0-9_]')  # Remove non-alphanumeric
    max_length = 32
    replacement_char = '_'
    to_lower_case = True

username = Username("John_Doe123")      # ✓ Valid
username = Username("john@doe!")        # Converts to "john_doe_"

Configuration Options: - allow_all_replacement_char: Allow strings of only replacement chars (default: True) - allow_empty: Whether empty strings allowed (default: True) - exact_length: Require exact length match (default: False) - max_length: Maximum string length (default: 512) - regex: Pattern for validation/replacement - regex_mode: REPLACE or MATCH mode - replacement_char: Character to replace invalid chars (default: '_') - strict_validation: Raise error instead of replacing (default: False) - to_lower_case: Convert to lowercase (default: False) - trim_whitespace: Trim leading/trailing whitespace (default: False)

Regex Modes:

from osbot_utils.type_safe.primitives.core.enums.Enum__Safe_Str__Regex_Mode import Enum__Safe_Str__Regex_Mode

# REPLACE mode: regex defines INVALID characters to replace
class SafeName(Safe_Str):
    regex = re.compile(r'[^a-zA-Z0-9]')  # Replace anything not alphanumeric
    regex_mode = Enum__Safe_Str__Regex_Mode.REPLACE

# MATCH mode: regex defines VALID pattern
class Version(Safe_Str):
    regex = re.compile(r'^v\d+\.\d+\.\d+$')  # Must match version pattern
    regex_mode = Enum__Safe_Str__Regex_Mode.MATCH
    strict_validation = True  # Required with MATCH mode

Using Primitives in Type_Safe Classes¶

Primitives integrate seamlessly with Type_Safe classes:

from osbot_utils.type_safe.Type_Safe import Type_Safe
from osbot_utils.type_safe.primitives.domains.identifiers.safe_str.Safe_Str__Id import Safe_Str__Id
from osbot_utils.type_safe.primitives.domains.numerical.safe_float.Safe_Float__Money import Safe_Float__Money
from osbot_utils.type_safe.primitives.domains.identifiers.Timestamp_Now import Timestamp_Now

class Order(Type_Safe):
    order_id: Safe_Str__Id
    amount: Safe_Float__Money
    created_at: Timestamp_Now

# Automatic validation on creation
order = Order(
    order_id="order-123",
    amount=99.99,
    created_at=Timestamp_Now()
)

# JSON serialization preserves types
json_data = order.json()
# {"order_id": "order-123", "amount": 99.99, "created_at": 1640000000000}

# Deserialization recreates typed objects
order2 = Order.from_json(json_data)
assert isinstance(order2.order_id, Safe_Str__Id)
assert isinstance(order2.amount, Safe_Float__Money)

Creating Custom Primitives¶

Custom Safe_Str¶

import re
from osbot_utils.type_safe.primitives.core.Safe_Str import Safe_Str

class Phone_Number(Safe_Str):
    regex = re.compile(r'[^0-9\-\+\(\) ]')  # Only phone chars
    max_length = 20
    trim_whitespace = True

phone = Phone_Number("+1 (555) 123-4567")

Custom Safe_Int¶

from osbot_utils.type_safe.primitives.core.Safe_Int import Safe_Int

class Age(Safe_Int):
    min_value = 0
    max_value = 150
    allow_str = True  # Allow "25" -> 25

age = Age(25)
age = Age("30")  # Automatic conversion

from osbot_utils.type_safe.primitives.core.Safe_Float import Safe_Float

class Temperature_Celsius(Safe_Float):
    min_value = -273.15  # Absolute zero
    max_value = 1000.0
    decimal_places = 1
    use_decimal = True

temp = Temperature_Celsius(20.5)

Best Practices¶

  1. Choose the right primitive: Use domain-specific types (Safe_Str__Email, Safe_UInt__Port) over generic ones
  2. Leverage type annotations: Use primitives in Type_Safe classes for full validation
  3. Handle errors gracefully: Primitives raise ValueError with descriptive messages
  4. Use strict_validation wisely: Enable for critical data, disable for flexible input
  5. Consider performance: use_decimal=False for high-performance float operations
  6. Document constraints: Custom primitive classes are self-documenting

Performance Considerations¶

  • Primitive instantiation: One-time cost at object creation
  • use_decimal=True: ~2-3x slower than native float, but exact
  • Regex validation: Cost depends on pattern complexity
  • String sanitization: Replace mode faster than match mode
  • Type_Safe integration: Minimal overhead with caching
